Physics Validation Module — backend_new/physics/

Deterministic neuro-symbolic checkers. Three independent validators + a 0–4 scorer + a human-readable explainer. Files: equation_validator.py · dimension_checker.py · numerical_validator.py · physics_scorer.py · new_checker.py · physics_explainer.py.

Validation chain (as invoked by the pipeline on every answer)

Input

Answer text (LaTeX-stripped by the pipeline first) — model output or deterministic EXPLORE/SWEEP response.

▼

1 · EquationValidator.validate(text) → (lhs_expr, rhs_expr, msg)

a_candidate_equations(): lines starting "Equation:" get priority, then every other = line; de-duplicated, order preserved.
b_clean_candidate(): skip prompt/evidence header lines (_SKIP_PATTERNS), strip LaTeX wrappers and "Label:" prefixes, cut trailing prose clauses ("… where J is …", new sentences).
cProse rejection _looks_like_prose(): any English connective word (per/of/is/the/…, ~45-word frozenset) or two consecutive ≥4-letter words on the RHS → candidate discarded. Stops "Cox = oxide capacitance per unit area" being scored as an equation.
dnormalize_equation(): LaTeX → ASCII (\frac{a}{b}→(a)/(b), \sqrt, \cdot…), Greek aliases (µ/μ→mu, γ→gamma, λ→lam), subscript variants (V_th/VT/VTn→Vth), SPICE notation (vDS→Vds), 70B habits (e**{x}→exp(x), Jo[expr]→Jo*(expr)), compound symbols (kT→k*T), ^→**.
eSymPy parse_expr on both sides with local_dict=self.symbols (~50 predeclared symbols) + implicit_multiplication_application. Candidates ranked by (known_symbols, −unknown_symbols); best wins.

Returns [OK] Equation parsed successfully or [WARN] No equation detected / Parsing failed.

▼lhs_expr / rhs_expr (SymPy) flow into checks 2 and 3

2 · DimensionChecker.check_equation(lhs, rhs)

Recursive AST walk (evaluate()) over base dimensions I, V, L, T, K, C:

  • Symbol → dim_map lookup (~50 entries; unknown = dimensionless)
  • Number/Rational/Half → {} (dimensionless)
  • Mul → exponents added (_mul_dims)
  • Pow → exponents scaled by float exponent (handles sqrt = Pow(x, 1/2))
  • Add → all terms must match, else {"DIMENSION_MISMATCH":1}
  • Function (log, ln, exp) → {} by convention

[OK] Dimensionally consistent: {dims} when simplified LHS dims == RHS dims, else [FAIL] Dimension mismatch. Example entries: mu = L²·V⁻¹·T⁻¹, Cox = I·T·V⁻¹·L⁻², gamma = V^0.5.

3 · NumericalValidator.evaluate(lhs, rhs)

Multi-point realism check at three self-consistent technology nodes — 100 nm, 22 nm, 5 nm — each a full SI parameter dict (~70 symbols + _COMMON_EXTRAS).

  • Substitute the node dict into RHS; indexed variants fall back (gm5→gm, go1/GL→gds).
  • LHS symbol looked up in RESULT_RANGES (per-quantity [lo, hi], e.g. Id ∈ [1e-15, 5e-2] A, ro ∈ [50, 1e9] Ω); unknown LHS → generic [1e-30, 1e15].
  • Majority vote: ≥ 2 of 3 nodes in range → [OK]; all unresolved → [WARN]; else [FAIL] Out of physical range.
  • Special case — constant RHS (e.g. C*E*h = 3): LHS evaluated at the nodes must match the constant within 3 orders of magnitude at ≥2 nodes, otherwise FAIL ("likely extracted number").
  • RHS = 0: passes only if LHS ≈ 0 (<1e-10) at ≥2 nodes.

The 0–4 physics score

physics_scorer.score_text(text) — Tier-2 fair scoring (single source of truth)

total = parseable(1.0) + dimensional(1.0) + numerical(1.0) + coverage_frac(0..1)
  • Coverage is partial: fraction of RHS free symbols with a known test value (NumericalValidator.test_values = the 22 nm node) — previously all-or-nothing.
  • Sub-checks computed independently from the parsed expression (no more string-coupling through num_msg).
  • strip_latex() applied first so all callers score identical normalized text.
  • Module-level singletons of the three validators (stateless, cheap).
  • Returned dict: parseable, dimensional, numerical, coverage (bool, =full), coverage_frac, total, equation, sym_msg, dim_msg, num_msg.

new_checker.py — fixed re-scorer (Stage-1 re-evaluation)

  • Same 0–4 shape (score_text / score_equation), plus:
  • _is_letter_soup() — rejects OCR/prose artifacts parsed into single-letter symbol products;
  • expanded device-physics vocabulary; Greek pre-normalization (_greek_pre);
  • _guard_trivial_dimensionless() — blocks trivially "consistent" dimensionless garbage;
  • separate coverage for the dimension step (_coverage_for_dim).
  • Used by stage1_physics_new.py, stage2, stage4 re-scoring — original physics_scorer left untouched for comparability.

Explainer

physics_explainer.py

explain_physics_score() and explain_compact(symbolic, dimensional, numerical, equation, model_label) convert the raw [OK]/[WARN]/[FAIL] messages into the human-readable verdicts shown in the frontend Inspector panel (api_server._make_explanation() is the only runtime caller). Helpers parse the dimension dict and unresolved-symbol lists back out of the message strings.
