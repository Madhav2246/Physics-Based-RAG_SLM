End-to-End Flow — one question, one pass

The complete life of a query, exactly as executed by RAGPipeline.answer() behind POST /api/query. Single spine, three-way branch at generation, everything reconverges into the same validation gauntlet.

1User asks a question — SLM Console (React, :3000)

Q&A Playground textarea + optional mode override (AUTO / LOOKUP / EXPLORE / SWEEP). apiFetch() → POST http://localhost:8000/api/query with {question, mode}. detail

▼

2FastAPI receives — api_server.rag_query()

Pipeline singleton (loaded once at startup, ~30 s) runs on a 2-worker ThreadPoolExecutor so the event loop stays free. Call: _pipeline.answer(question, force_mode). detail

▼

3Hybrid retrieval — top-3 evidence chunks

  • Dense: bge-large-en-v1.5 (1024-d, query prefix, cosine via FAISS IndexFlatIP) → 6 candidates
  • Sparse: BM25, physics-aware tokenizer (Vgs-Vth → two tokens) → 6 candidates
  • RRF fusion (k=60) → top 6 → CrossEncoder rerank (ms-marco-MiniLM-L-6-v2) → top 3

detail

▼

4Mode decision + corpus equation extraction

  • force_mode honored if valid, else detect_mode(): parseable range → SWEEP; design phrase ("what W/L…") → EXPLORE; else LOOKUP
  • prefer_vars (design modes): the equation must contain the variables being solved/plotted
  • _find_corpus_equation() scans all chunks: SymPy-parseable, ≥2 symbols, letter-soup guard, prose rejected
  • Design-query retrieval retry on a cleaned query if no covering equation; node profile detected ("5nm GAA")
  • Honesty guard: design mode with no governing equation → relabelled LOOKUP

detail

▼branch on mode
5 · ANSWER COMPOSITION — three lanes, one survives

SWEEP

Solve symbolically once (SymPy), lambdify over linspace(start, stop, 50), matplotlib PNG (family-of-curves supported). No SLM math.

any failure ▼ falls to EXPLORE

EXPLORE

Extract user values (regex + verified-SLM stage) → solve(Eq(lhs,rhs), target) → physical branch pick → substitute with provenance tracking → derivation audit text.

any failure ▼ falls to LOOKUP

LOOKUP (default & final fallback)

Prompt = evidence + corpus equation → Qwen2.5-0.5B+LoRA generates 3 seeded samples → each scored 0–4 by the validators → best wins (cosine tiebreak). Answer = Equation: … + explanation.

explore/sweep detail · SLM detail

▼one response string continues (always starts "Equation: …")

6Validation gauntlet — every answer, no exceptions

  • LaTeX sanitise — strip \[ \] \( \) $, map \frac → / so SymPy can parse
  • Symbolic — EquationValidator.validate(): real parseable equation? (~50-symbol dictionary)
  • Dimensional — recursive AST unit walk: LHS units ≡ RHS units?
  • Numerical — evaluate at 100 nm / 22 nm / 5 nm nodes, LHS-specific ranges, ≥2/3 majority vote
  • Semantic similarity — cosine(answer, evidence) catches structurally-valid-but-wrong answers

detail

▼

7Confidence + stability

  • LOOKUP → compute(): retrieval .20 + symbolic .15 + dimensional .20 + numerical .20 + length .10 + similarity .15
  • EXPLORE/SWEEP → score_explore(): corpus .25 + unique solution .20 + dims .10 + sanity .07 + parse .03 + value provenance 0–.25
  • Labels: ≥0.80 HIGH · ≥0.55 MODERATE · else LOW — REVIEW REQUIRED
  • Stability: agreement across the 3 samples = 1 − (unique−1)/(total−1)
▼

8Audit log

RAGLogger appends the full record — query, evidence, response, all three validator verdicts, similarity, confidence, stability — to logs/rag_logs.jsonl (UTC ISO-8601).

▼

9JSON back to the console

{ response, confidence, breakdown[], symbolic, dimensional, numerical,
  stability{score,label}, similarity, chunks[], explanation{per-check verdicts},
  mode, explore_result{provenance[], sweep{x,y}}, node_profile, sweep_plot_url }

Rendered as the Confidence Inspector: verdict cards, evidence chunks, provenance table, sweep <img>. Thumbs-down → HITL correction → LoRA retraining loop. detail

Fallback chain: SWEEP → EXPLORE → LOOKUP — the system always answers, and the returned mode always tells the truth about which path actually ran.
