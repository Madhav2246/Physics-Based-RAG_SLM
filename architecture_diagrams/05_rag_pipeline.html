RAG Pipeline Orchestrator — backend_new/pipeline/rag_pipeline.py

Class RAGPipeline — the 10-step flow behind every /api/query. One instance owns all components (constructed once at API startup; the SLM is shared with the extractor via exploration_engine.set_slm_model(self.slm) — no duplicate weights).

Members built in __init__

retriever = HybridRetriever() · slm = TinySLM() · validator = EquationValidator() · dimension_checker = DimensionChecker() · numerical_validator = NumericalValidator() · logger = RAGLogger() · confidence_engine = ConfidenceEngine() · uncertainty_engine = UncertaintyEngine() · exploration_engine = ExplorationEngine(validator.symbols) · node_manager = NodeProfileManager()

answer(query, force_mode=None) → dict — step by step

1Retrieve evidence — retriever.retrieve(query, top_k=cfg.TOP_K=3) (hybrid dense+BM25+RRF+rerank, see 01_retrieval).
▼
2Mode — force_mode (UI button) if ∈ {LOOKUP, EXPLORE, SWEEP}, else detect_mode(query). Then compute prefer_vars: SWEEP → {target_var, sweep_var} (+ family param); EXPLORE → {detected target}. These bias equation selection toward equations containing the variables the user wants.
▼
3Corpus equation extraction — _find_corpus_equation(evidence, prefer_vars): scans every candidate in every chunk via validator._candidate_equations() + validate(), applies the _acceptable() guard (real expressions on both sides, not 2=3, ≥2 symbols, letter-soup rejection — needs a multi-char symbol or nearly all symbols recognised). LOOKUP path prefers candidates with ≥2 recognised physics symbols; SWEEP/EXPLORE path prefers an equation whose free symbols cover prefer_vars (fallback = first acceptable). This guard is the fix for the Stage-1/4 coverage-zero bug (prose like "current density" used to parse into garbage like I = N*d*e*i**2*j*r*t).
3bRetrieval retry (design queries only) — if SWEEP/EXPLORE found no equation or one not covering prefer_vars: _clean_design_query() strips "using 5nm GAA node", plot/sweep/versus keywords and numeric ranges, re-retrieves, and takes the retry equation only if it covers the wanted vars (or nothing existed before). LOOKUP never retries.
3cNode profile — node_manager.detect_from_query() → as_tracker_defaults(node); None ⇒ 100 nm baseline. Honesty guard: SWEEP/EXPLORE with no governing equation is relabelled LOOKUP so the returned mode reflects what actually ran.
▼three-way branch on mode

SWEEP

  • parse_sweep_request() + parse_multi_curve_param(); sweep_req.node_name = detected node or 100nm_CMOS.
  • Family param present (and ≠ target/sweep var) → solve_sweep_multi() → plot_multi() PNG; else solve_sweep() → plot() PNG under data/evaluation/sweep_*.png.
  • Any error / empty curve / exception → fall back to EXPLORE (single point).

EXPLORE (also SWEEP's fallback)

  • exploration_engine.solve(lhs, rhs, query, node_defaults) — deterministic SymPy derivation + audit (see 04_exploration_design).
  • Error → silent fallback to LOOKUP.
  • Response comes from format_response(); if it doesn't already start with "Equation:", the derived form is prepended (validator-compatible format).

LOOKUP (default / final fallback)

  • build_prompt(query, evidence, corpus_equation) → slm.generate_multiple(n=3).
  • Best-of-N neuro-symbolic re-ranking: each candidate is LaTeX-stripped, then scored (sym_ok + dim_ok + num_ok + cov_ok) ∈ 0..4; ties broken by cosine(candidate, evidence) using the dense embedder.
  • Winner promoted to responses[0]; final text = "Equation: {corpus_eq | NOT FOUND IN CORPUS}\n\n{winner}".
▼single merged path continues
4LaTeX sanitise — _strip_latex() removes \[ \] \( \) $$, maps \frac→/, \cdot→*, strips backslash commands (0.5B ignores anti-LaTeX instructions; raw LaTeX crashed SymPy on subscripts).
5Symbolic validation — validator.validate(response) → (lhs, rhs, symbolic_validation).
6Dimensional validation — dimension_checker.check_equation(lhs, rhs); skipped with [WARN] if no parse.
7Numerical validation — numerical_validator.evaluate(lhs, rhs); skipped with [WARN] if no RHS.
8Semantic similarity — cosine(embed(response), embed(joined evidence)) via retriever.dense.embed_model; catches structurally-valid but factually-wrong answers. Non-fatal on failure (None).
▼

▼
10Uncertainty — uncertainty_engine.evaluate(responses): normalize (strip+lower), agreement = 1 − (unique−1)/(total−1); ≥0.8 HIGH / ≥0.5 MODERATE / else LOW STABILITY. Logging — RAGLogger.log() appends the full record (query, evidence, response, all three validations, similarity, confidence, uncertainty) to logs/rag_logs.jsonl with UTC ISO-8601 timestamp.
