FastAPI app "Physics RAG SLM API" v1.0.0 · uvicorn 0.0.0.0:8000 · CORS allow-all · static mount /static/evaluation → data/evaluation/ (sweep PNGs for the frontend).

Startup & environment

  • os.chdir(backend_new/) so all relative paths in utils/config.py resolve; sys.path gets the project root.
  • HF_HOME = <repo>/hf_cache computed from file location (portable across drives — old hardcoded d:/S6/... broke). If hf_cache/hub exists → HF_HUB_OFFLINE=1, TRANSFORMERS_OFFLINE=1; otherwise downloads allowed.
  • Broken SSL_CERT_FILE env entries removed if the file no longer exists.
  • Pipeline singleton: @app.on_event("startup") → _load_pipeline() on a ThreadPoolExecutor(max_workers=2): constructs RAGPipeline(), then retriever.dense.load_index() and retriever.sparse.build_index_from_docs(dense.documents). Sets _pipeline_ready=True (~30 s first run). Queries before that return {"error": "Pipeline loading…"}.
  • NVIDIA NIM: NVIDIA_API_KEY from env (never hardcoded — removed in commit 6afcc1fa), base URL https://integrate.api.nvidia.com/v1, baseline model meta/llama-3.3-70b-instruct.

Concurrency model

Inference

CPU-bound pipeline calls run via loop.run_in_executor(_executor, …) on the shared 2-worker ThreadPoolExecutor, keeping the asyncio event loop free. rag-compare parallelises its two NIM calls the same way.

Long-running jobs

Synthesis / evaluation / training run as separate Python subprocesses (subprocess.Popen) so they can't block or crash the server; progress is exchanged through small live JSON files the GET endpoints poll.
