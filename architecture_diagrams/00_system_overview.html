Pipeline singleton loaded in background thread at startup (ThreadPoolExecutor(max_workers=2)). 17 endpoints: query / baseline (NVIDIA NIM 70B) / rag-compare (NIM Qwen 1.5B & 3B) / registry / ingest / synthesis / evaluation / feedback / train. Serves sweep PNGs at /static/evaluation. Detail: 06_api_server

▼_pipeline.answer(question, force_mode) via executor

Orchestrator — pipeline/rag_pipeline.py → RAGPipeline.answer()

10-step flow: retrieve → mode detect (LOOKUP / EXPLORE / SWEEP) → corpus-equation extraction → generate or solve deterministically → 3 physics validators → semantic similarity → confidence → uncertainty → JSONL log. Detail: 05_rag_pipeline

▼owns one instance of each component below

Retrieval — retrieval/

  • HybridRetriever: RRF(k=60) fusion
  • DenseRetriever: FAISS + BAAI/bge-large-en-v1.5 (1024-d, cosine/IndexFlatIP)
  • SparseRetriever: BM25Okapi, physics-aware tokenizer
  • CrossEncoderReranker: ms-marco-MiniLM-L-6-v2

01_retrieval

Reasoning — reasoning/

  • TinySLM: Qwen2.5-0.5B-Instruct + LoRA adapter (models/finetuned_slm)
  • N=3 samples, temp 0.3, top_p 0.9, max 128 new tokens, SEED 42
  • prompt_builder.build_prompt(): 2 templates (with / without corpus equation)

02_reasoning_slm

Physics validation — physics/

  • EquationValidator — extract + SymPy parse
  • DimensionChecker — recursive AST unit analysis
  • NumericalValidator — 3 tech nodes, majority vote
  • physics_scorer / new_checker — 0–4 score
  • physics_explainer — Inspector text

03_physics_validation

Design synthesis — physics/

  • ExplorationEngine — SymPy solve, no SLM math
  • SweepEngine — parametric sweeps + matplotlib PNG
  • ValueTracker — user/corpus/default provenance
  • NodeProfileManager — 5nm GAA / 16nm FinFET / 28nm FDSOI / 100nm CMOS profiles
  • SLMExtractor — 2-stage value extraction (stage 1 dormant)

04_exploration_design

Utils — utils/

  • ConfidenceEngine — weighted score (compute / score_explore)
  • UncertaintyEngine — agreement across N samples
  • RAGLogger — append-only JSONL audit
  • config.py — single source of model/retrieval constants

Offline / batch lane

Scripts — backend_new/scripts/ (also mirrored in kaggle_regen/, kaggle_sweep/ for GPU runs)

  • Ingestion: ingest.py CLI → _ingestion_engine.IngestionEngine (PyMuPDF → 384/64-word chunks → FAISS + BM25 + registry)
  • Data synthesis: synthesize_data.py — NIM-generated golden QA (target 100: 40 easy / 40 medium / 20 hard)
  • Training: train_slm.py, train_from_feedback.py (HITL 70/30 mix → candidate adapter), evaluate_candidate.py (promotion gate)
  • Report evaluation: stage1…stage6 + sweep_retrieval.py — physics score, generation quality, retrieval Hit@k/MRR/NDCG, ablations, significance

Detail: 08_evaluation_training

External services

  • NVIDIA NIM (https://integrate.api.nvidia.com/v1, key from NVIDIA_API_KEY env): meta/llama-3.3-70b-instruct (no-RAG baseline), qwen/qwen2.5-1.5b-instruct & qwen/qwen2.5-3b-instruct (RAG size comparison), plus golden-QA synthesis.
  • Hugging Face Hub — only when hf_cache/hub missing; otherwise fully offline (HF_HUB_OFFLINE=1).
