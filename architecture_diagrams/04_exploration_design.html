Exploration & Sweep Module — backend_new/physics/

Deterministic design synthesis: no SLM call for any math. Every answer = corpus equation + SymPy algebra + explicit value provenance. Files: exploration_engine.py · sweep_engine.py · value_tracker.py · node_profile_manager.py · slm_extractor.py.

Mode detection — module function detect_mode(query)

  • SWEEP checked first (most specific): fires only when parse_sweep_request(query) actually parses a numeric range — a bare "plot" without a range falls through.
  • EXPLORE — any of ~20 trigger phrases (_EXPLORE_TRIGGERS): "how do i", "what w/l", "design the", "calculate the", "what aspect ratio", "what oxide thickness", …
  • LOOKUP — default safe path (SLM explains from evidence).
  • UI can force any mode via force_mode in RAGPipeline.answer(); invalid/AUTO → auto-detection.

EXPLORE flow — ExplorationEngine.solve(lhs, rhs, query, node_defaults)

1 · detect_target(query)

12 regex TARGET_PATTERNS (WL, tox, Cox, NA, gamma, gm, Vth, Vth0, Id, ro, Phi_f, Vov). Key rule: a symbol given with an explicit value ("gamma = 0.4") is a known input, excluded from target matching — otherwise "What Vth with gamma = 0.4?" would wrongly target gamma.

▼

2 · extract_user_values(query, node_defaults) → ValueTracker (two-stage)

S1SLM stage (SLMExtractor.extract): constrained JSON prompt to the already-loaded TinySLM (n=1, 120 tokens). Keys restricted to 14 _VALID_SYMBOLS; every value must pass the verification gate _verify_extractable() — traceable to a numeric token in the query within 2% relative tolerance (after unit-word conversion: milliamp, half, nm…). Only finite values block stage 2. Empirically dormant: benchmark (June 2026) — regex-only 20/22, two-stage 20/22; the 0.5B returns {} in practice. Architecture kept so a proper NER model can drop in.
S2Regex stage: ~30 VALUE_PATTERNS — structured (Id = 1mA, tox: 2nm) and natural language ("2 milliamps", "Vgs of 0.9 volts", "overdrive voltage is half", "gate length of 100 nanometers"). Regex deliberately overwrites stage 1 when it matches, because it converts units to SI correctly (0.5B often extracts "500" from "500 microamps").
▼

3 · solve_for(lhs, rhs, target, tracker, corpus_eq_str) — never raises, always returns a dict

  1. WL special case: corpus stores W and L separately → substitute W → WL·L so SymPy cancels L and returns e.g. WL = 2·Id/(mu·Cox·(Vgs−Vth)²).
  2. sympy.solve(Eq(lhs, rhs), target); empty solution list → error "target does not appear in the corpus equation".
  3. Branch pick _pick_physical_branch(): multiple roots → evaluate each numerically, keep those inside SANITY_RANGES, choose smallest |value| (physical threshold root); fallback to solutions[0].
  4. Vov→Vgs mapper (deterministic precedence): explicit user Vgs wins; else user Vov ⇒ Vgs = Vth + Vov registered as a derived user value so the audit log matches the computation.
  5. tracker.resolve(free_symbols) → subs() → float(evalf()); NaN/Inf and unresolved symbols become explicit errors.
  6. Sanity check against SANITY_RANGES (e.g. WL ∈ [0.5, 10000], tox ∈ [0.3nm, 200nm], NA ∈ [1e12, 1e22]).
▼

SWEEP flow — solve_sweep() / solve_sweep_multi() + SweepEngine

Query grammar — parse_sweep_request()

  • Gate: query must contain plot / sweep / versus / vs / range / from.
  • Range: from X[unit] to Y[unit] or range X to Y; units via _UNIT_SCALE (pA…kV…mm).
  • Two grammars: with "from", the variable immediately before it sweeps ("W/L versus Vov from 0.1 to 0.6" → Vov); with "range", the first of the vs-pair sweeps.
  • Aliases (_canonical_var): w/l·aspectratio→WL, overdrive→Vov, draincurrent→Id, lambda→lam…
  • → SweepRequest(sweep_var, target_var, start, stop, n_points=50, node_name).
  • parse_multi_curve_param(): "for Id = 0.5mA, 1mA, 2mA" → family parameter + SI values (first token's unit inherited by bare numbers; ≥2 values required).

Execution — SweepEngine.run_sweep()

  • Reuses the symbolic solution from solve_for() — never re-solves.
  • Collision guard: sweeping the target itself → error → pipeline falls back to EXPLORE.
  • Vov axis fix: if sweeping Vov and the expression contains Vgs → substitute Vgs = Vov + Vth (mirrored in multi).
  • Everything except the sweep symbol substituted from the tracker; leftover free symbols → explicit error.
  • sympy.lambdify(..., modules="numpy") over np.linspace(start, stop, 50); scalar results broadcast.
  • plot()/plot_multi(): matplotlib Agg (headless) → PNG under data/evaluation/sweep_*.png, axis units from _VAR_UNIT, provenance note box ("mu, Cox, tox, Vth: 5nm_GAA defaults"), plasma colormap + legend for families.

Value provenance & node profiles

ValueTracker — single source of numeric truth

  • TrackedValue(symbol, value, unit, provenance, description); provenance ∈ user corpus default.
  • Class DEFAULTS: 21 entries, 100 nm baseline (mu 0.05, Cox 0.02, Vth 0.4, k, q, T…). Constants k/q tagged corpus.
  • resolve(needed) fill order: instance defaults (node profile) → class DEFAULTS safety net.
  • Three consumers: solve_for() (get_subs_dict()), ConfidenceEngine.score_explore() (provenance_fraction = user count / total), audit log (audit_lines()).

NodeProfileManager

  • JSON profiles in physics/node_profiles/: 5nm_GAA, 16nm_FinFET, 28nm_FDSOI, 100nm_CMOS (default).
  • detect_from_query(): ordered keyword scan — "gate-all-around"/"gaa"/"5nm" → 5nm_GAA; "finfet"/"16nm"; "fdsoi"/"28nm"; "100nm". No keyword → None → generic baseline.
  • as_tracker_defaults(node): copy 100 nm DEFAULTS, overlay the profile's params (each tagged provenance=default, description "[profile] note") — physical constants never lost.
