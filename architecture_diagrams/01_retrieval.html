Retrieval Module — backend_new/retrieval/

Hybrid dense + sparse retrieval with Reciprocal Rank Fusion and cross-encoder reranking. Files: hybrid_retriever.py · dense_retriever.py · sparse_retriever.py · reranker.py. All constants from utils/config.py.

Query-time flow — HybridRetriever.retrieve(query, top_k=cfg.TOP_K)

Input

query: str · top_k defaults to cfg.TOP_K = 3 (caller may override; pipeline passes it explicitly).

▼fan-out: both retrievers asked for top_k × 2 = 6 candidates each

Dense — DenseRetriever.retrieve(query, top_k=6)

1Prepend query prefix: "Represent this sentence for searching relevant passages: " (cfg.QUERY_PREFIX, required by bge-large)
2Encode with SentenceTransformer BAAI/bge-large-en-v1.5 (1024-d), normalize_embeddings=True
3faiss.IndexFlatIP.search() — inner product on normalized vectors = cosine similarity
4Return docs in FAISS ranked order (explicitly no set() — ordering preserved)

Sparse — SparseRetriever.retrieve(query, top_k=6)

1Tokenize with physics-aware regex _TOKEN_RE: identifiers [A-Za-z][A-Za-z0-9_]* + numbers incl. scientific notation → "Vgs-Vth" becomes ["Vgs","Vth"], 1e-7 kept whole
2BM25Okapi.get_scores(tokens) over pre-tokenised corpus
3Sort indices by score desc, return top-k documents
▼ ▼two ranked lists merge

Reciprocal Rank Fusion (in HybridRetriever.retrieve)

For each list: rrf_scores[doc] += 1 / (k + rank) with k = 60 (standard RRF constant). Docs appearing in both lists accumulate both contributions. Sort by RRF score desc, keep top_k × 2 = 6 candidates.

▼6 fused candidates

Rerank — CrossEncoderReranker.rerank(query, docs, top_k)

Model: cross-encoder/ms-marco-MiniLM-L-6-v2. Builds (query, doc) pairs → model.predict(pairs) → sort by score desc → return top 3 document strings.

▼

Output

list[str] — 3 evidence chunks, passed to prompt builder / corpus-equation extractor.

Design decisions recorded in code comments

  • Determinism fix: candidate merge uses ranked dict accumulation, not set() union — set ordering is hash-dependent and made runs non-reproducible.
  • Ranking fix: dense results previously went through list(set(...)) which destroyed FAISS distance ordering; now order-preserving.
  • Tokenizer fix: BM25 previously used doc.split(), so "Vgs-Vth" was a single token and keyword matches failed on hyphenated physics symbols.
  • Revert path: to restore the old MiniLM index set EMBED_MODEL_NAME="all-MiniLM-L6-v2", EMBED_DIM=384, QUERY_PREFIX="", NORMALIZE_EMBEDDINGS=False.
