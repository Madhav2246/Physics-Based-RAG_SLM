## 💡 The idea

LLMs hallucinate equations with total confidence. A 70B model will happily tell you
`I = V/R` governs MOSFET saturation. Instead of scaling the model up, we scale the
**verification** up:

> **Small model + hybrid retrieval + deterministic physics checkers > big model on trust.**

Every answer — whether generated by our fine-tuned **Qwen2.5-0.5B**, or by a **70B NIM
baseline** we run for comparison — is passed through three independent validators:

| Check | Engine | Question it answers |
|---|---|---|
| 🧮 **Symbolic** | SymPy parser + ~50-symbol physics dictionary | Is there a real, parseable equation here? |
| 📏 **Dimensional** | Recursive AST unit analysis (I, V, L, T, K, C) | Do both sides carry the same physical units? |
| 🔢 **Numerical** | Evaluation at **three real technology nodes** (100 nm / 22 nm / 5 nm), majority vote | Does it produce physically plausible values? |

No check can be smooth-talked. `Cox = oxide capacitance per unit area` is prose, not an
equation — rejected. `C*E*h = 3` evaluates fine but matches nothing physical — rejected.
A dimensionally broken equation from a 70B model — **caught, flagged, scored down.**

## 🚀 What it does

### Three answer modes, auto-detected from your question

| Mode | Trigger | What happens |
|---|---|---|
| 🔍 **LOOKUP** | *"What is the saturation drain current equation?"* | Hybrid retrieval → equation extracted **from the corpus** (not the model) → SLM only explains the symbols → best-of-3 candidates re-ranked by physics score |
| 🎯 **EXPLORE** | *"What W/L do I need for Id = 1 mA with Vov = 0.5 V?"* | **Zero SLM math.** SymPy rearranges the corpus equation, substitutes your values, and prints a full derivation audit with the provenance of every number (`user` / `corpus` / `default`) |
| 📈 **SWEEP** | *"Plot W/L versus Vov from 0.1 to 0.6 for Id = 0.5mA, 1mA, 2mA"* | Solves symbolically **once**, vectorizes with NumPy, renders trade-off curves (families supported) as PNG |

### Process-node awareness
Say *"using 5nm GAA"* and the solver swaps in that node's parameter profile
(`5nm_GAA`, `16nm_FinFET`, `28nm_FDSOI`, `100nm_CMOS`) — constants like k, q, T are never lost.

### Honest confidence, not vibes
A weighted confidence score with a full per-component breakdown in the UI:
retrieval quality, symbolic parse, dimensional check, numerical realism, response length,
and semantic similarity between answer and evidence. EXPLORE mode swaps similarity for
**value provenance** — answers built from your numbers score higher than answers built from defaults.
Plus a stability score across 3 sampled generations (all seeded — results are reproducible).

### Human-in-the-loop training
Bad answer? Correct it in the UI → corrections accumulate as JSONL → one click fine-tunes a
**candidate LoRA adapter** (70/30 correction-to-base mix against catastrophic forgetting) →
a promotion gate evaluates it before it can replace the live adapter.

## 🖥️ The console

A React 18 + TanStack Router + shadcn/ui frontend ("SLM Console") with four pages:

- **Q&A Playground** — side-by-side *ungrounded Llama-3.3-70B* vs *our grounded 0.5B*, with a

- **Evaluation** — launch golden-QA synthesis and full pipeline evaluation, watch live progress, real confidence histograms
- **Model Tuning** — pending corrections, before/after preview, hyperparameters, live training loss

## 📊 Evaluation (6-stage, statistically tested)

Scored on an NVIDIA-NIM-synthesized golden QA set (100 questions: 40 easy / 40 medium / 20 hard)
with a model-agnostic **0–4 physics score** = parseable + dimensional + numerical + symbol-coverage.

| Stage | What it proves |
|---|---|
| **1 — Physics validation** | Our grounded 0.5B vs raw 0.5B vs ungrounded 70B, same scorer for all |
| **2 — Generation quality** | BERTScore, ROUGE-L, BLEU-4, faithfulness, answer relevancy |
| **3 — Retrieval** | SOTA sweep picked **bge-large @ 384/64 chunks**; live hybrid path (dense+BM25+RRF+rerank) confirmed with anchor-based Hit@k / MRR / NDCG@3 |
| **4 — Ablation** | Remove reranker / sparse / dense / best-of-N / validator one at a time — each earns its place |
| **5 — Token sensitivity** | Score vs generation budget |
| **6 — Significance** | Wilcoxon signed-rank + effect sizes on every headline claim |

All generation is seeded (`SEED=42`); every raw answer is stored so future scorers can
re-evaluate **without re-generation**. Kaggle-ready mirrors live in
[`kaggle_regen/`](kaggle_regen/) and [`kaggle_sweep/`](kaggle_sweep/).

## ⚡ Quickstart

```bash
# ── Backend ─────────────────────────────────────────────
cd backend_new
pip install -r requirements.txt
uvicorn api_server:app --host 0.0.0.0 --port 8000
# first start takes ~30s: loads Qwen + LoRA, FAISS index, BM25

# ── Frontend ────────────────────────────────────────────
cd frontend_new
npm install
npm run dev          # → http://localhost:3000, talks to :8000

# ── Or just the CLI demo ────────────────────────────────
cd backend_new && python main.py
```

Optional: set `NVIDIA_API_KEY` (see `.env.example`) to enable the 70B baseline
and model-size comparison panels.
